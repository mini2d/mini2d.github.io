Creating the example game - part 2

In the previous part we created the menu state and now it is time for us to finish the game and write the gameState game state.

Creating the gameState

Once again, we begin by calling the addState function. This time, however, specifying both the init and update function.

var gameState = game.addState({
    init: function(){
        // more code to come
    },
    update: function(){
        // more code to come
    }
});

That should already feel familiar from the previous chapter, so let's proceed!

The init function

Most of the code is going to be in the init function, the engine will automatically take care of most of the tasks we need. The update function will be pretty straight forward, but more on that later, lets finish the init function first.

Adding the paddles

We begin by adding the paddles. This is almost identical code two times over, with the exception of the position of the paddles.

game.paddle1 = this.addSprite(50, 50, 'assets/paddle.png');
game.paddle1.enableBody();
game.paddle1.body.collideWorldBounds();
game.paddle1.body.immovable = true;

game.paddle2 = this.addSprite(500, 50, 'assets/paddle.png');
game.paddle2.enableBody();
game.paddle2.body.collideWorldBounds();
game.paddle2.body.immovable = true;

Each paddle is represented by a sprite, which is simply created with the addSprite function. The arguments specify its position and a texture file.

Because we want to take advantage of the built in physics system of mini2d.js, we have to call the enableBody method on both of these sprites. This will create a body based on the sprite's properties.

The next thing we are going to want to do, is to call the collideWorldBounds method, which will ensure, that the paddles remain inside the game bounds.

Last, but not least, we set the immovable property to true, this will cause the physics engine not to translate the sprite on impact.

Adding the ball

Now that we created a pair of paddles, we need a ball for our pong game, so let's create it.

game.ball = this.addSprite(game.width / 2, game.height / 2, 'assets/ball.png');
game.ball.enableBody();
game.ball.body.velocity.x = 250;
game.ball.body.velocity.y = 100;
game.ball.body.bounce = 1;
game.ball.body.collideWorldBounds(function(direction){
    if(direction == 'left'){
        game.player2points += 1;
        game.player2score.text = "Player 2 score: " + game.player2points;
        game.ball.body.position.x = game.width / 2;
        game.ball.body.position.y = game.height / 2;
    }
    else if(direction == 'right'){
        game.player1points += 1;
        game.player1score.text = "Player 1 score: " + game.player1points;
        game.ball.body.position.x = game.width / 2;
        game.ball.body.position.y = game.height / 2;
    }
});

This introduces a few more methods and properties. The velocity property simply tells the physics engine to move the ball accordingly every frame.
The bounce property means how much will the velocity be reduced upon impact (0 being nullifying the velocity, 1 having no impact on velocity).

Then we have a familiar method collideWorldBounds. This time it has arguments though. The argument is a callback function to be called upon impact, which supplies a direction parameter in the form of string, which specifies what side of the world did the body collide with.

The world collision callback then modifies score properties we haven't declared yet. So let's fix that.

game.player1points = 0;
game.player2points = 0;
game.player1score = this.addText(0, 10, 'Player 1 score: 0', '15px sans-serif', '#000');
game.player2score = this.addText(450, 10, 'Player 2 score: 0', '15px sans-serif', '#000');

The first two lines are just a variable for us to store the score.

The two following lines take advantage of the Text class, to create a score counter on the top of the screen.

The addText method should feel familiar, because we did something similar when adding text to the button. The only difference here is that the first two arguments specify the text's position, which was unnecessary for the button, because we wanted the text to be centered, which is handled automatically by the mini2d.js game engine.

The update method

Now that we have the init method ready for our game, it is time to write the update method as well.

In this method there are two things that we want to do.

  • Position the paddles according to the current position of the mouse.
  • Make the ball collide with the paddles.

Positioning the paddles

So first things first, let's position them.

game.paddle1.body.position.y = game.input.y - game.paddle1.body.height / 2;
game.paddle2.body.position.y = game.input.y - game.paddle2.body.height / 2;

We simply set the position according to the game input, furthermore, we substract the half of the height of the paddle from the number, so that our cursor is vertically in the middle of the paddle.

Paddle-ball collision

So now we want to make the ball collide with the paddles, right? So let's do just that!

game.paddle1.body.collide(game.ball.body);
game.paddle2.body.collide(game.ball.body);

And just like that we achieved the very thing we desired to achieve.

This concludes our tutorial and you should now understand how the Pong example works.
